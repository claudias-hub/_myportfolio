How I like to work

  • Start with clarity: define the goal, the users and the main flows before adding features.
  • Focus on basics first: clean layout, readable text, good contrast and working navigation.
  • Build in small steps: ship something that works, then refine based on feedback.
  • Keep it understandable: prefer clear names, simple structures and small, focused components.
  • Make it maintainable: use Git, small commits and straightforward deployment setups.

You can explore some examples of this approach on my Projects.
